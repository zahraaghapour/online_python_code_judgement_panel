Solve given problem regarding notations

Problem:

Ben is a milkman. He is responsible to take and selling the milk portion of some houses in a region of Newyork city. Every morning he should visit the number of houses. But he has a problem. He does not know how to manage his path to houses and arrange houses to visit. He wants to have totally shortest path. With respecting these conditions:

  • He wants to visit all houses. Also, he does not like backing again to previously visited houses. Of course, at the end, he wants to go back to the starting house, because the starting house is his neighbor.
  • He knows the starting house and the position of each house.
Help Ben to Find an optimum path.

Details:

For implementation use 'TSP' class:

TSP class:

Each member of this class has 3 attributes. They will initialize in __init__ function.
  1. start : Shows the index of startting house.

  2. position : Which is a dictionary type. Keys are indexes assigned to each house. The values of each key are the position of that house in a coordinate plane. The type of values is 'tuple'. like this one{1:(x,y)}.

  3. vertex_num : Shows the number of houses which should be visited. Is not as an input argument for '__init__'. You can use the 'position' input to value this attribute.


this class has 4 methods:
  1. calc_distance : This attribute gets two houses indexes as an input and returns euclidean distance of two houses.

  2. sequence : When we call this method(without input) it will return a set which includes all sequences which are possible to arrange houses. Sequences are tuple type, which first element is the index of the starting house and the next elements show intermediate houses to visit and the last element is the same as the starting house. like this: {(1,..,1),(1,..,1),..}

    3. notations:

    Note that this problem is an NP-hard problem. So in large spaces, it is not easy to solve with this solution and we will need stochastic solutions. In this case, we consider small space to our problem, therefore, we can calculate the number of possible solutions or sequences. '(vertex_num-1)!' is the number of all possible sequences.

  3. seq_distance : This attribute gets a sequence with tuple type and returns the total distance between houses mentioned in sequence from starting one to ending one.

  4. find_min : This attribute gets a set of tuple sequences and selects a solution with less path distance. And returns minimum length of distance and the path related respectively.
